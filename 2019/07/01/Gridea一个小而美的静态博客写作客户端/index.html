Gridea 一个小而美的静态博客写作客户端

发表于 2019-07-01 | 分类于 开源项目 | 阅读数

前言

当下大多数人可能已经对 Github Pages 不再陌生，他是很多喜欢写文章的人第一次接触的用于免费搭建 Blog 的工具，使用 Github Pages 你能很快速的构建一个简洁却又不失优雅的个人 Blog ，直到现在互联网上还有大量的基于 Github Pages 的个人博客和搭建方法的教程。然而搭建虽然简单，但是管理和上传文章却相对麻烦不少，官方的教程里，你需要经历繁琐的 Git 步骤才能发布一篇文章和修改你的个人空间界面。对于那些想更便捷和优雅的操作 Github Pages 写作者来说，我郑重推荐这款工具 Gridea 。

配置

Gridea 最早叫 Hve Notes ，开发者为了更易读和好记，重新命名为 Gridea，支持 Windows 和 Mac 平台，他的基础界面非常的小清新。

第一次使用他需要你进行应用的初始化配置，才能让他和 Github Pages 连接。配置很简单，可以参考下面的方法进行配置：

  • 域名：Github Pages 对应的域名（例如：https://brick713.github.io 或 http://moyu.life）
  • 仓库：你的静态文件存放的仓库（例如：http://Brick713.github.io）
  • 分支：你的 Github Pages 对应的分支（例如：master）
  • 用户名：仓库所属用户的用户名（通常就是你的 Github 账号的昵称）
  • 邮箱：Git 推送时使用的邮箱（通常就是你的 Github 账号的邮箱）
  • Token: Git 推送时需要的 Token，用来向 Github 提交构建后的文件 Github personal access tokens
  • CNAME: 可通过这个选项配置你自己的域名（例如：moyu.life）

这里有个小 Tips，当你建立你自己的 Personal access tokens 的时候，建议你只需要赋予 repo 的权利就可以了，这样可以减少你的 Token 出现泄漏后可能造成的风险。

美化

配置好了，检测远程连接没有问题之后，Gridea 就成功和你的 Github Pages 连接了。重新回到主界面，点击左下角的同步，就可以将你的文章同步到你的个人 Blog 了，你可以访问你的网站来看看效果。

接下来你可以自定义一些配置让你的个人博客更加个性化。在配置界面里，你可以增加你的 Blog 的评论系统，目前 Gridea 支持 Gitalk 和 Disqus，这里就不详细介绍如果配置评论系统，有需求的同学可以查看作者给出的小白教程，你可以修改你的 Favicon 和头像配置。

Gridea 也提供了 Blog 的主题设置，在这里你可以选择你的主题，目前 Gridea 支持4 款主题 ，分别是『Notes』、『Simple』、『Fly』、『Paper』，你也可以修改网站名称、底部信息等其他主题内容。

如果你对 Blog 的主题有自己的理解和审美，那么你可以在自定义配置中自行修改相关的样式和配色，包括布局、颜色和你的自定义样式，同样你也可以在你的 Blog 中展示你的其他社交账号，并且使用谷歌统计来查看你的博客访问情况，这里也希望作者能结合国内的情况，在未来的版本增加百度统计。

写作

Gridea 支持 Markdown 语法并且最新的版本也支持了 Katex 公式，对于一个普通的写作者来说，他满足了你最基本的写作需求，在文章的创作界面中，作者为了写作的方便，也提供了一些基础 Markdown 语法的快捷操作，例如插入代码块、链接、图片，改变字体等等，通过点击那些显而易见的图标能够帮助提升你的写作速度，节约你的时间。

在写作界面的右边栏，你可以给你的文章命名一个好记的 URL、选定一个相关主题的标签、修改你的创建时间或者给你的文章配一个贴切的封面图。

如果你的一篇文章只创作了一半，你不想让他出现在你的 Blog 中，但你又修改了其他的文章的内容需要同步到你的 Github Pages 上，那么你可以先将你的文章勾选为列表中隐藏，他可以将你的文章变更为 Hide 状态，这样这篇文章就不会被上传，而已经修改过的文章则会照常同步到 Blog 上，如图所示：

这个人性化的功能也方便你更好的管理你的文章，可以隐藏掉一些文章却不需要删除他们。其他诸如添加标签和管理主页 Tag 的功能也可以根据你的个人喜好自行修改。

总结

Gridea 是一款麻雀虽小却五脏俱全的静态博客写作客户端，他能让你告别繁琐的 Git 操作，更加快速方便的管理你的个人 Blog，同时 Gridea 恰到好处的主题和实用的配置功能也能一定程度上帮助你美化 Blog，如果你的动手能力够强，甚至可以自己调教。作为一款免费的工具，如果你有任何的建议都可以去官方项目页面进行反馈，根据目前的版本迭代速度看，作者还是非常勤劳的，期待未来有更多的新鲜功能。来自：Brick713的小黑屋的文章。
